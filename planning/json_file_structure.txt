Phase 1: Define the Target Schema First (Before Cleaning)

Do not clean JSON blindly. First define how the data should look in PostgreSQL.

Identify Core Entities:

1. members
2. bills
3. amendments
4. votes
5. vote_records



Phase 2: Design Relational Model (Conceptual Level)

members:
- member_id (PK)
- full_name
- first_name
- last_name
- party
- state_name
- chamber
- bio fields
- age
- years_in_congress



bills:
- bill_id (PK)
- congress
- chamber
- title



amendments
- amendment_id (PK)
- bill_id (FK)
 - description



votes
- vote_id (PK)
- legislation_type
- legislation_num
- chamber
- congress
- session
- vote_date
- result



vote_records
- vote_id (FK)
- member_id (FK)
- position



Phase 3: Clean & Normalize JSON Structure

Reshape JSON files to reflect normalized entities.

EXAMPLE: votes.json
[
  {
    "vote_id": 123,
    "bill_id": "HR123",
    "date": "2026-01-14",
    "result": "Passed",
    "total_yes": 60,
    "total_no": 40
  }
]




Phase 4: Cleaning Checklist

1. Ensure Consistent Keys
2. Standardize Data Types
3. Remove Null Noise
4. Generate Stable IDs



Phase 5: Validate JSON Structure

Load each JSON into pandas & check:
- Duplicate primary keys
- Missing foreign keys
- Orphan vote_records
- Unexpected nulls



Phase 6: Prepare for PostgreSQL

1. Write SQL CREATE TABLE scripts
2. Load JSON via Python (psycopg2 / SQLAlchemy / SQLite)
